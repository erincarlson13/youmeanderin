DETAILS

Every data analyst has been on the receiving end of a frantic email from the IT team after submitting a large, complex, and/or server-crushing query. It happens. We need to ensure that the appropriate governance is in place to limit this sort of interaction.

In order to do this, the following steps will be necessary:

  • Define a set of intelligent default governing settlngs for end users, with the possibility for administrator override
  • Surface a variety of server governing controls to administrators
  • Expose a set of query statistics and functionality so that business analyst can gauge the impact and performance of his/her queries
  • Augment the alerting capabilities to ensure administrators are aware of long-running queries

USER STORY

As an administrator, I want to define a set of query size limit controls so that my users are running better-performing queries that serves the double purpose of not stressing the underlying enterprise resources.

DETAILS

In order to do this, we should surface a setting in the Server Administration properties dialog that empowers administrators specify how many total rows that they feel comfortable sending to the database server. If exceeded, the error message ought to instruct users to contact their administrators to increase the number of allowed rows.

Product Roadmap Planning

PORTFOLIO ITEM

Governed Data Discovery | How to Ensure a Win-Win Outcome that IT Administrators, BI Users, and Business Analysts can Agree On

As a BI administrator, I want to know that I will be able to provide a best-in-class experience to my data analysts while balancing the needs of the IT department, so that the power of data discovery is balanced by system- and IT-defined governance.

DETAILS

The proliferation of desktop data analysis tools connecting directly to enterprise resources frequently results in stressed or failing IT resources. Customers frequently cite scenarios where servers are brought down by adhoc queries made against at-capacity database servers as the most common problem. The ultimate result of these hardware failure scenarios is additional stress on system administrators who are expected to triage such situations, service disruptions for BI users using standard reports, and frustration with respect to performance on the business analyst side.

In order to support all of these user personas, we need to find ways to effectively and intelligently govern adhoc access to enterprise assets while still advocating for a powerful adhoc user experience.
